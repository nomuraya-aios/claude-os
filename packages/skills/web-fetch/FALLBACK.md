# Web Fetch Skill - Fallback

## エラーパターン

### Jina Readerアクセス失敗

**症状**: `https://r.jina.ai/` 経由でのアクセスがタイムアウトまたはエラー

**対処**:
1. ユーザーに状況を説明
2. 素のURLでのリトライを提案
3. 承認を得てから実行

**出力例**:
```
⚠️ Jina Reader経由でのアクセスに失敗しました。

エラー: [エラーメッセージ]

素のURLで再試行しますか？
（LLM向け最適化が行われないため、トークン消費が増える可能性があります）
```

### URLが無効

**症状**: URLのフォーマットが不正

**対処**:
1. ユーザーに正しいURLの提供を依頼
2. URLの例を示す

**出力例**:
```
❌ URLが無効です。

正しい形式で入力してください。

例:
- https://example.com/article
- https://github.com/user/repo
```

### プロンプトが不明確

**症状**: ユーザーの意図が不明

**対処**:
デフォルトプロンプトを使用して実行

```
プロンプトが指定されていないため、以下のデフォルトプロンプトで実行します：
「このページの内容を要約してください。主要なポイント、キーワード、重要な情報を日本語で抽出してください。」
```

## デバッグ情報

スキル実行時に以下を記録：

- 元URL
- Jina URL（プレフィックス追加後）
- 使用したプロンプト
- WebFetch実行結果（成功/失敗）
- エラーメッセージ（ある場合）
